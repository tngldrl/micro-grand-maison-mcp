# Micro Grand Maison MCP - MCP (AI Analysis Server)

Vertex AI (Gemini, Imagen 3) を活用して、Git リポジトリの静的解析、アーキテクチャの合成、およびアバター生成と透過処理を行う AI 解析サーバー。

## 🌟 主な機能

* **静的ソースコード解析とメタデータ抽出 (Phase 1 & 2)**:
  * クローンされたリポジトリ内のコードチャンクから、API ルート定義、DB スキーマ、依存関係、使用テクノロジーをスキャンします。
  * `ThreadPoolExecutor` (マルチスレッド) により、大量のチャンクファイルを並行して高速に個別解析します。
* **論理アーキテクチャの合成 (Phase 3)**:
  * Gemini 3.5 Flash を使用して、各コードチャンクの解析結果を一つの論理的なシステムグラフへと統合・重複排除します。
  * システムの接続密度やディレクトリ構造を基に、2D 配置用のレイアウトパターン（`hierarchical`（階層）, `radial`（放射状）, `mesh`（環状）など）を算出し、座標メタデータを割り当てます。
  * 生成されたシステム説明文は、指定されたキャラクター設定（レストランの料理人やサービススタッフのアナロジー）にマッピングされ、自然な日本語に変換されます。
* **アバター画像生成と HSV アルファ透過処理 (Phase 4)**:
  * 各サービスのために **Imagen 3** を用い、デコラティブな衣装をまとったちびキャラ（ポケモン風タッチ）アバターとチャットアバターを並行生成します。
  * **高精度な透過処理**:
    * 背景に指定されたクロマキー用の「純粋な緑色」を検出し透過させます。
    * 生成画像のエッジにある「緑の影やくすみ」「反射光」に対しても、RGB 値を **HSV (Hue, Saturation, Value) 色空間** に変換して判定するロジックを実装。色相 (Hue) が 70〜170 度の範囲にあるピクセルを高精度に検出して透過（BFS境界平滑化）するため、不自然な緑のフチが残りません。
* **キャッシュバスティング**:
  * アバター画像の同一ファイル名上書きロードによるブラウザキャッシュ問題を回避するため、返却されるアバター URL の末尾に UNIX タイムスタンプパラメータ（`?t=timestamp`）を付与します。

## 📂 主要ディレクトリ・コード構造

```
micro-grand-maison-mcp/
├── server.py          # FastAPI エンドポイント (/analyze, /chat)、アバター画像 HSV 透過処理、並行処理ハンドラー
├── scanner.py         # ソースコードファイルのチャンク分割、依存関係ファイルのスキャン処理
├── analyzer.py        # Gemini を用いたスケルトン抽出、統合アーキテクチャの合成と座標マッピング、チャット推論
├── generator.py       # Google Gen AI SDK (Vertex AI) を用いた Imagen 3 画像生成と自動再試行（リージョン間フェイルオーバー）
├── output/
│   └── avatars/       # ローカルに生成されたアバター画像 (.png) の静的格納用フォルダ
└── requirements.txt   # Python 依存関係一覧
```
